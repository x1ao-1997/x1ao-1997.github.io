使用

元素创建表单的方式

  • 页面中的
    元素相互独立
  • 每一个
    元素应该至少附带两个属性：action method
  • 元素也可以附带所有的通用属性、UI事件属性以及下面的属性：
  • enctype accept accept-charset onsubmit onreset
属性 说明
action 表明提交表单时将如何处理数据。通常action属性的值是Web服务器中的一个页面或者一个程序。大多数浏览器将只接受作为action属性值的以http://开头的URL。
method 将表单数据发送给服务器的方法（get和post）
id 允许用户唯一标识页面中的
元素
onsubmit 在浏览器中运行一段脚本，在表单被发送给服务器之前检查所输入数据的正确性。
onreset 清空表单中的所有内容，它的值是一个脚本，当用户单机调用该脚本的按钮时执行它。
enctype 指定浏览器编码数据的方式1.application/x-www-form-urlenconded(默认)2.multipart/form-data(上传文件或非ASCII字符)
accept-charset 制定一个字符编码列表（用户可以输入该列表）然后服务器根据该列表进行处理。
accept 类似accept-charset属性，不同之处是它采用一个以逗号隔开的内容类型（或文件类型）列表该属性的思想是用户不能上传与列表中所列举的内容类型不同的文件
target 通常用于元素中，指明新的页面将被加载到哪一个框架或浏览器窗口中。
